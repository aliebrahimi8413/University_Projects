# بازی تعادل پاندول معکوس

یه پاندول (میله) که روی یه گاری وایساده و باید تعادلش حفظ بشه که نیفته.
یا خودت با کلید چپ/راست گاری رو کنترل کن، یا بذار کامپیوتر خودش تعادلش
رو نگه داره.

## اجرا

```
python3 main.py
```

نیاز به tkinter داره (معمولا همراه پایتون نصبه).

## فایل‌ها

- `pendulum.py` -> کلاس Pendulum (زاویه و سرعت زاویه‌ای میله)
- `cart.py` -> کلاس Cart (مکان و سرعت گاری)
- `controller.py` -> کلاس Controller (تصمیم میگیره چقدر نیرو بزنه، دستی یا خودکار)
- `simulation.py` -> کلاس Simulation (فیزیک رو حساب می‌کنه، چک می‌کنه باختی یا نه)
- `gui.py` + `main.py` -> رابط گرافیکی با tkinter

## فیزیک

فرمول حرکت پاندول-روی-گاری رو از منابع کلاسیک درس کنترل (مسئله‌ی معروف
Cart-Pole) گرفتم. برای حل معادلات از روش اویلر ساده استفاده کردم (ساده‌ترین
روش انتگرال‌گیری عددی: هر بار شتاب رو حساب می‌کنی و با یه گام زمانی کوچیک
سرعت و بعد مکان رو آپدیت می‌کنی).

## کنترلر خودکار چطور کار می‌کنه

فرمولش اینه:

```
نیرو = k1×زاویه + k2×سرعت‌زاویه‌ای + k3×مکان‌گاری + k4×سرعت‌گاری
```

یعنی هرچی میله بیشتر کج بشه یا سریع‌تر کج بشه، یا گاری از وسط دور بشه،
نیروی بیشتری برای اصلاح می‌زنیم. ضریب‌ها (۵۰، ۱۰، ۱، ۱) رو با امتحان
چندتا مقدار مختلف پیدا کردم تا دیدم کدومش میله رو نمی‌ندازه (این روش
بهش state feedback میگن).

## نمره‌ی اضافه: نویز سنسور

با تیک زدن "نویز سنسور روشن باشه"، کنترلر به‌جای زاویه‌ی واقعی، یه زاویه‌ی
کمی اشتباه (با نویز گاوسی، پیش‌فرض انحراف‌معیار ۲ درجه) می‌بینه - دقیقا
مثل یه سنسور واقعی که هیچ‌وقت ۱۰۰٪ دقیق نیست. تست کردم و کنترلر تا حدود
۵ درجه نویز هم می‌تونه تعادل رو حفظ کنه، بعدش شروع می‌کنه به لرزیدن بیشتر.

## یه باگ که موقع تست پیدا شد

اولش وقتی موقع بازی دستی یه کلید رو نگه می‌داشتی و بعد دکمه‌ی "شروع دوباره"
رو می‌زدی، کلید هنوز "فشرده" حساب می‌شد (چون `reset()` مجموعه‌ی
`keys_pressed` رو پاک نمی‌کرد)! در نتیجه بعد از ریست هم گاری همچنان هل
داده می‌شد. این رو با تست واقعی (شبیه‌سازی فشردن کلید و بعد ریست) پیدا
کردم و با پاک کردن `keys_pressed` توی `reset()` درستش کردم.

## چیزایی که میشه بهش اضافه کرد

- سختی بازی رو بیشتر کنیم (مثلا میله بلندتر یا سبک‌تر، تعادلش سخت‌تره)
- امتیاز و رکورد بهترین زمان
- استفاده از Pygame به‌جای tkinter برای گرافیک بهتر
